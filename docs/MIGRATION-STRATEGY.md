# Migration Strategy: Zero Friction Adoption

## Core Principle

> **vibe-state-cli is additive. It never deletes, overwrites, or replaces anything the user already has.**

## What Users Already Have (Must Not Touch)

| User's existing files | vibe's behavior | Status |
|---|---|---|
| `.claude/skills/*.md` | **Never touch** — user's custom skills | v0.1.0: Fixed |
| `.claude/CLAUDE.md` (hand-written) | **Append**, not overwrite | TODO v0.2.0 |
| `.cursorrules` or `.cursor/rules/` | **Add new file**, not overwrite existing | OK |
| `AGENTS.md` (hand-written) | **Append vibe section**, not overwrite | TODO v0.2.0 |
| `.windsurf/rules/`, `.clinerules/`, `.roo/rules/` | **Add new file** | OK |
| Any `.env`, `.gitignore`, `pyproject.toml` | **Never touch** | OK |

## What vibe Adds (New, Non-Conflicting)

| vibe creates | Conflicts with? | Strategy |
|---|---|---|
| `.vibe/` directory | Nothing — unique to vibe | New directory |
| `.vibe/state/*.md` | Nothing | New files |
| `.vibe/VIBE.md` | Nothing | New file |
| `.vibe/config.toml` | Nothing | New file |
| `.claude/rules/vibe-standards.md` | User might have other rules | Unique filename, no conflict |
| `.cursor/rules/vibe-standards.mdc` | User might have other rules | Unique filename, no conflict |
| `AGENTS.md` | User might have one | **Must merge, not overwrite** |
| `CLAUDE.md` | User might have one | **Must merge, not overwrite** |
| `GEMINI.md` | User might have one | **Must merge, not overwrite** |

## v0.2.0 TODO: Smart Merge for Root-Level Files

For `CLAUDE.md`, `AGENTS.md`, `GEMINI.md`:

1. If file does not exist → create it (current behavior, correct)
2. If file exists and was generated by vibe (has `<!-- vibe-state-cli:integrity:... -->`) → safe to overwrite
3. If file exists and was NOT generated by vibe → **append vibe section at bottom**, wrapped in markers:

```markdown
<!-- vibe-state-cli:start -->
## Vibe Workflow
(vibe-generated content here)
<!-- vibe-state-cli:end -->
```

On `vibe adapt --sync`, only the content between markers gets updated.

## User Communication

For the README / onboarding:

> vibe-state-cli works alongside your existing setup. It won't touch your custom CLAUDE.md,
> .cursorrules, or skills. It only adds a `.vibe/` directory and optional adapter files
> with unique names (like `vibe-standards.md`) that don't conflict with your existing rules.
